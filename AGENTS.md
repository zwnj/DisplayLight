# DisplayLight 開発ガイド

## このリポジトリの目的

DisplayLightは、Windowsのディスプレイ消灯設定と、手動で切り替える一時的なスリープ防止を扱うWPF常駐アプリです。
製品の設計は`DESIGN.md`、未決事項は`docs/OPEN_QUESTIONS.md`を正とします。

## 作業前に読むもの

- 仕様や状態遷移を変える場合は`DESIGN.md`を読む。
- 未決事項に触れる場合は`docs/OPEN_QUESTIONS.md`を読み、決定を推測で確定しない。
- 長い作業や複数フェーズの作業では`PLANS.md`に沿って実行計画を作る。
- Windows電源APIやCodex Hookの仕様は、実装前に公式資料で現在の挙動を確認する。

## リポジトリ構成

- `src/DisplayLight.App`：WPF UI、アプリ起動、Windowsアダプター、設定保存、依存関係の組み立て。
- `src/DisplayLight.Core`：UIとWin32 APIに依存しない状態、ポリシー、ユースケース。
- `tests/DisplayLight.Core.Tests`：Coreの高速な単体テスト。
- `tests/DisplayLight.App.Tests`：実電源操作を行わないWindowsアダプターとViewModelのテスト。
- `docs`：検証方法、未決事項、Architecture Decision Record（ADR）。
- `scripts`：ローカルとCodexから共通利用するセットアップおよび検証スクリプト。

## 標準コマンド

初回セットアップを行う。

```powershell
pwsh -NoProfile -File ./scripts/setup.ps1
```

変更を検証する。

```powershell
pwsh -NoProfile -File ./scripts/verify.ps1
```

アプリを起動する。

```powershell
dotnet run --project ./src/DisplayLight.App/DisplayLight.App.csproj
```

`pwsh`がない環境では、`powershell -NoProfile -ExecutionPolicy Bypass -File ...`を使用してよい。

## アーキテクチャ規約

- CoreからWPF、Win32、`Process`、ファイルシステム、通知ライブラリへ依存させない。
- OS操作、時計、バッテリー取得、通知、設定保存、IPCはインターフェース越しに扱う。
- UIのコードビハインドには表示固有の処理だけを置き、判断ロジックを置かない。
- 複数セッションの集約はCoreで行い、「一つでもWorkingなら防止する」という規則を一か所に保つ。
- 外部イベントはそのままドメイン状態にせず、アダプターで検証して正規化する。
- 新しい本番NuGet依存は、標準ライブラリで代替できない理由と保守上の影響を明示してから追加する。

## 電源操作の安全規約

- 自動テストからディスプレイ設定変更、スリープ、休止、シャットダウンを実行しない。
- `powercfg`とWin32 APIはモック可能なアダプター内だけで呼び出す。
- スリープ防止要求は、失敗、アプリ終了、利用者の解除時に解放できる設計にする。
- ユーザーが明示したスリープ操作は妨げない。
- 自動スリープは既定で無効にし、実機検証でもカウントダウンとキャンセル手段を省略しない。
- エラー時は電源要求を保持し続けず、通常のWindows電源管理へ戻す。
- 実機の電源状態を変える確認は、実行内容と復旧手順を先にユーザーへ示す。

## データとログ

- ソースコード、プロンプト、エージェント回答、ファイル内容、認証情報を保存または送信しない。
- ログへ残せるのは、状態名、匿名化したセッション識別子、時刻、結果コードなど運用に必要なメタデータだけとする。
- 作業ディレクトリは既定でログに残さない。
- 設定ファイルの破損時は安全な既定値へ戻し、元ファイルを診断用に残す方針とする。

## テスト方針

- 状態遷移、電源条件、設定値の境界はCoreの単体テストで扱う。
- OSアダプターは引数、終了コード、エラー変換をテストし、実電源操作は手動テストへ分離する。
- バグ修正には、可能な限り先に失敗を再現するテストを追加する。
- 手動確認が必要な変更では`docs/TESTING.md`の該当チェック項目と結果を報告する。

## 完了条件

- 要求された振る舞いと`DESIGN.md`が一致している。
- `pwsh -NoProfile -File ./scripts/verify.ps1`が成功している。
- 電源操作や状態遷移を変えた場合は、対応する自動テストがある。
- 利用者が見る挙動を変えた場合はREADMEまたは設計文書を更新している。
- 未実施の実機確認は、実施済みのように書かず明記している。
- 最後に差分を見直し、無関係な変更、秘密情報、生成物が含まれていないことを確認している。

## ファイルと文章

- テキストファイルはUTF-8で扱う。
- PowerShellで読み書きする場合は`-Encoding UTF8`を明示し、`>`と`>>`で書き込まない。
- 公開型と複雑な安全判断には、「何をするか」ではなく「なぜ必要か」が分かる説明を残す。
- ADRで確定した判断を変更する場合は、既存ADRを書き換えず、新しいADRで置き換える。
