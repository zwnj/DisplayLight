# 検証方針

## 自動検証

すべての変更で次を実行します。

```powershell
pwsh -NoProfile -File ./scripts/verify.ps1
```

このスクリプトは、パッケージ復元、書式確認、Releaseビルド、単体テストを順に実行します。
CIもWindowsランナーで同じ検証段階を実行します。

## 自動テストで禁止する操作

- PowrProf APIによる利用者の実電源設定変更
- Power Requestの実ハンドル取得
- スリープ、休止、シャットダウン
- 自動起動の登録と解除
- 利用者の`%LOCALAPPDATA%`にある実設定の上書き

自動テストはネイティブAPIのテストダブルか、一時ディレクトリを使用します。

## MVPの自動テスト

Coreでは次を検証します。

- `1分`、`5分`、`10分`、`30分`、`60分`、`無期限`と秒数の対応
- プリセット外の値を近い値へ暗黙に変更しない表示方針
- 手動要求、AC限定、現在の電源状態から物理的な要求を決める規則
- バッテリー切り替え時に利用者の要求を保ったまま物理的な要求を解放する規則
- 設定スキーマと不正な列挙値の正規化
- CoreがWPF、Win32、ファイルシステムへ依存しないこと

Appでは次を実電源操作なしで検証します。

- PowrProf APIのAC値とDC値の読取り順序
- 対象側だけを書き込み、電源プランを反映してから両方を読み直すこと
- 読戻し不一致を成功としないこと
- プリセット外の現在値を表示し、保存済みスライダー選択を維持すること
- 片側を適用しても、反対側の未適用スライダー選択を維持すること
- 初期化が終わるまで操作コマンドを無効にすること
- 設定の同一ディレクトリ内一時保存と置換
- 壊れたJSONを診断用ファイルへ残し、安全な既定値へ戻ること

## MVPの実機確認

電源状態を変える項目は、変更前の値、実行内容、復旧手順を利用者へ先に示してから行います。
2026年7月15日の実施結果は[`docs/verification/2026-07-15-mvp.md`](./verification/2026-07-15-mvp.md)に記録します。

### 起動と通知領域

- [x] Windows 11で通知領域APIへの登録が成功する。
- [x] 画面の「終了」でプロセスが終了コード`0`を返す。
- [x] 二回目の起動が既存インスタンスを表示し、自身は終了する。
- [ ] Windows 10でアイコンが表示される。
- [ ] 左クリック、右クリック、キーボード選択が仕様どおり動く。
- [ ] DPI 100%、150%、200%でメニューが画面外へ出ない。
- [ ] Explorer再起動後にアイコンが再表示される。
- [ ] アプリ終了後に残骸アイコンが残らないことを目視する。

### ディスプレイ設定

- [x] 変更前のAC値とDC値を記録する。
- [x] AC値を変更し、ネイティブAPIの読戻しが一致する。
- [x] DC値を変更し、ネイティブAPIの読戻しが一致する。
- [x] 確認後に変更前のAC値とDC値へ復旧する。
- [ ] AC値だけの変更でDC値が変わらないことを実機で記録する。
- [ ] DC値だけの変更でAC値が変わらないことを実機で記録する。
- [ ] Windows設定画面で変更したプリセット外の値を再取得する。
- [ ] 権限エラーを成功表示しないことを制限ユーザーで確認する。
- [ ] Windows 10でも同じ値を読み書きできる。

### 手動スリープ防止

- [x] 画面操作からPower Requestを作成し、状態を有効と表示する。
- [x] 画面操作からPower Requestを解除し、状態を無効と表示する。
- [ ] `powercfg /requests`などOS側の一覧で要求の追加と削除を確認する。
- [ ] アイドルタイムアウトによるシステムスリープを防ぐ。
- [ ] 要求中でもディスプレイが設定時間で消灯する。
- [ ] 利用者が開始したスリープを妨げない。
- [ ] ACからバッテリーへの切り替えで要求を解放し、AC復帰で再取得する。
- [ ] 強制終了後に要求が残らないことをOS側で確認する。
- [ ] Modern StandbyのDC時制約を長時間確認する。

## Agent Preview以降の確認

次の項目はMVPの完了条件に含めません。

- Codex AppとCLIでのHookイベント観測
- 外部入力のスキーマ、サイズ、時刻、現在ユーザー制限
- 複数セッションの集約と欠落イベントの期限切れ
- 作業完了後のカウントダウン、キャンセル、自動スリープ
- アプリ未起動時の連携方式
