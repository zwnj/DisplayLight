# タスクトレイのコントロールセンターへの再設計

- 状態：Implemented（実機確認継続）
- 作成日：2026-07-15

## 目的

DisplayLightの主画面を、中央に開く通常ウィンドウから、通知領域の近くへ表示する小型コントロールセンターへ変更します。
利用者が状態確認、スリープ防止、現在の電源側の消灯時間変更を短い操作で完了できることを目標とします。

## 完了条件

- 左クリックで通知アイコンの近くにフライアウトが表示されます。
- 同じアイコンの再クリック、外側クリック、Escapeでフライアウトが閉じます。
- フライアウトはタスクバーとAlt+Tabへ通常ウィンドウとして表示されません。
- 幅は360から392 DIPの範囲で調整し、リサイズ不可、標準タイトルバーなしで表示されます。
- 一枚の半透明シェルと弱い内側カードにより、通常ウィンドウではなく通知領域フライアウトとして見えます。
- スリープ防止が最初の主要操作として表示されます。
- AC限定はスリープ防止へ従属する設定として表示されます。
- ACとバッテリーの現在値を確認でき、各プリセットはスライダー形状の離散選択で変更できます。
- 未適用の値がある行だけに小型の適用操作が表示されます。
- 成功通知は自動的に消え、エラーは発生箇所の近くに残ります。
- ライト、ダーク、高コントラストで文字と状態を識別できます。
- キーボードだけで開く、操作する、閉じることができます。
- NarratorがACとバッテリーを区別し、`無期限`を含む実際のプリセット名を読み上げます。
- 既存の電源API、安全ポリシー、設定形式を変更しません。
- 自動テストから実電源設定とPower Requestを操作しません。
- `pwsh -NoProfile -File ./scripts/verify.ps1`が成功します。

## 対象外

- Codexおよび他のエージェントとの連携
- 自動起動、インストーラー、更新
- 電源APIと設定スキーマの変更
- モニター単位のディスプレイ制御
- 独立した詳細設定ウィンドウ
- 製品ロゴと通知領域アイコンの刷新
- Windows 10への対応

## 目標レイアウト

```text
┌──────────────────────────────┐
│ DisplayLight      電源: AC  ⋯ │
├──────────────────────────────┤
│  スリープ防止                │
│  オフ                    切替 │
│  アイドルスリープだけを防止   │
│  AC電源接続時のみ       [ ]   │
├──────────────────────────────┤
│  ディスプレイ消灯時間          │
│  AC電源時       現在 10分  ▴ │
│   変更後 30分                │
│   1 ──●────── 無期限    適用  │
│  バッテリー時   現在 5分  ▾ │
├──────────────────────────────┤
│  必要な場合だけ通知を表示       │
└──────────────────────────────┘
```

現在の電源側を既定で展開し、反対側は一行表示にします。
一度に展開するスライダーは一つだけとし、利用者が反対側の行を選ぶと展開対象を切り替えます。
右上の補助メニューには再読込みと終了を置きます。

## 開閉仕様

手動起動した場合は、通知領域の初期化後にフライアウトを一度表示します。
左クリックはフライアウトの表示と非表示を切り替えます。
キーボード選択はフライアウトを表示し、スリープ防止トグルへフォーカスします。
右クリックは小さなショートカットメニューを表示し、「開く」と「終了」を提供します。
外側クリックとEscapeはフライアウトを閉じますが、アプリは終了しません。
終了は補助メニューまたは右クリックメニューからだけ実行します。

通知アイコンの画面座標は`Shell_NotifyIconGetRect`から取得します。
Microsoftは通知領域から開くポップアップをクリック位置の近くへ配置するよう案内しています。
参照：[Notification Area](https://learn.microsoft.com/en-us/windows/win32/shell/notification-area)、[Shell_NotifyIconGetRect](https://learn.microsoft.com/en-us/windows/win32/api/shellapi/nf-shellapi-shell_notifyicongetrect)

## ウィンドウ仕様

- `WindowStyle=None`、`ResizeMode=NoResize`、`ShowInTaskbar=False`を使用します。
- WPF `Popup`へ置き換えず、キーボードフォーカスとAutomationを扱える小型WPF `Window`を維持します。
- 恒久的な`Topmost=True`は使用せず、表示時だけ前面化します。
- 幅は372 DIPとし、六つのプリセットへ44 DIP程度の操作領域を確保します。
- 高さは内容に合わせ、モニターの作業領域から上下24 DIPを引いた値を上限とします。
- 配置は通知アイコンの外接矩形、タスクバーの辺、モニターの作業領域、モニター別DPIから計算します。
- 下タスクバーだけでなく、上、左、右の配置でも画面内へ収めます。
- Windows 11ではDWMの角丸と一時ウィンドウ向け背景効果を技術検証します。
- Windows 11でも背景効果を利用できないBuildや設定では、単色サーフェス、1px境界、システム影を使用します。

Windows 11の角丸指定はBuild 22000以降で利用できます。
一時ウィンドウ向けの`DWM_SYSTEMBACKDROP_TYPE`はBuild 22621以降で利用できます。
参照：[Rounded corners](https://learn.microsoft.com/en-us/windows/apps/desktop/modernize/ui/apply-rounded-corners)、[DWM_SYSTEMBACKDROP_TYPE](https://learn.microsoft.com/en-us/windows/win32/api/dwmapi/ne-dwmapi-dwm_systembackdrop_type)

## 情報と操作の仕様

### BatteryMonitorから採用するデザイン原則

[zwnj/BatteryMonitor](https://github.com/zwnj/BatteryMonitor)を、WPF通知領域フライアウトの視覚的な参照実装とします。
一枚の半透明サーフェス、小さい製品名、一つの主状態、境界線を抑えた内側カードを採用します。
比較の詳細は[BatteryMonitorのデザイン参照レビュー](../reviews/2026-07-15-battery-monitor-reference.md)に記録します。

ホバー表示、ピン留め、ドラッグ移動、手動テーマ切替、メトリクス中心のカード構成はMVPへ含めません。
これらは消灯設定の操作を助けず、開閉状態と設定項目を増やすためです。

### ヘッダー

製品名、現在の電源状態、補助メニューだけを44から52 DIPの範囲へ置きます。
製品名は14 DIP程度の補助表示とし、スリープ防止の状態を18 DIP程度の主表示にします。
現在のキャッチコピーと大型の濃紺ヘッダーは削除します。

### スリープ防止

スリープ防止は全幅のトグルタイルとして先頭へ置きます。
状態は`オン`、`オフ`、`AC接続待ち`を文言で表示し、色だけで伝えません。
AC限定はタイルの直下に従属するスイッチとして置きます。

### ディスプレイ消灯時間

ACとバッテリーを一つのセクションへまとめ、それぞれを弱いトーンの独立カードとして表示します。
各行は対象名、現在値、展開状態を表示します。
展開した行だけに離散プリセットの選択と適用操作を表示します。
選択値が現在値と異なる場合だけ「変更後」と適用操作を表示します。
ドラッグ中はWindows設定へ書き込まず、選択値が現在値と違う場合だけ「適用」を表示します。

見た目はスライダーを維持しますが、Automation上は`1分`から`無期限`までの名前付き選択肢として公開します。
標準SliderのAutomationで実値を正確に公開できない場合は、ListBoxまたはRadioButtonをスライダー形状へスタイルし、SelectionItemとして各プリセット名を公開します。

六つのプリセットは等間隔の停止位置として扱います。
通常時は現在値と選択値が一致するため、16 DIP程度のつまみを一つだけ表示します。
未適用の変更がある場合は、現在値の位置へ小さな輪郭リングを残し、変更後の位置をアクセント色のつまみで示します。

トラックは2 DIP程度、未選択の目盛りは4から6 DIPを基準とします。
フォーカスはつまみの外側へ2 DIPのリングとして表示し、フォーカス時にレイアウトを動かしません。
各停止位置には実際のプリセット名を常時表示し、左右キーで一つずつ移動できる44 DIP程度の操作領域を確保します。
選択操作だけではWindowsへ書き込まず、「適用」で確定します。

### 通知

成功はPoliteなライブリージョンへ短時間表示します。
失敗は該当セクションの近くへ残し、再試行できる場合だけ操作を表示します。
固定フッターは設けません。

## 実装境界

Coreのプリセット、電源ポリシー、設定モデルは変更しません。
Appへ通知アイコンの矩形取得、フライアウト配置、開閉状態を扱うWindows固有アダプターを追加します。
配置計算は画面座標と作業領域を入力する純粋な計算へ分離し、実機を使わずに単体テストできるようにします。
ViewModelには展開中の電源側、未適用状態、セクション別エラー、一時通知を追加します。

## 実装手順

### Phase 1：ウィンドウシェルと配置

1. 通知アイコンの矩形を取得するアダプターを追加します。
2. タスクバーの四辺と複数モニターに対応する配置計算を追加します。
3. 標準タイトルバーとタスクバー表示をなくした固定幅フライアウトを作ります。
4. 左クリック、再クリック、外側クリック、Escape、単一インスタンス表示要求を新しい開閉処理へ接続します。
5. 配置計算と表示切替の自動テストを追加します。

### Phase 2：情報階層とコンパクト表示

1. 大型ヘッダー、三つの独立カード、常設フッターを削除します。
2. スリープ防止タイルを先頭へ移します。
3. AC限定をスリープ防止へ従属させます。
4. ACとバッテリーを一つの消灯時間セクションへ統合します。
5. 現在の電源側だけを展開し、未適用時だけ小型の適用操作を表示します。
6. 右クリックメニューを補助操作へ縮小します。
7. 製品名を補助表示へ下げ、スリープ防止の状態を画面内の主表示にします。
8. 内側の強い枠線を減らし、余白と背景トーンで領域を分けます。
9. 離散スライダーへ現在値リング、変更後のつまみ、固定プリセット名を実装します。

### Phase 3：テーマとアクセシビリティ

1. 固定色を意味別のDynamicResourceへ置き換えます。
2. ライト、ダーク、高コントラストの各テーマを定義します。
3. プリセット選択を実値名でAutomationへ公開します。
4. キーボード起動時の初期フォーカスと、閉じた後の`NIM_SETFOCUS`を実装します。
5. 見出し階層、ライブリージョン、装飾要素のAutomation設定を追加します。
6. 主要操作へ44×44 DIP程度の操作領域を確保します。

### Phase 4：Windows 11の外観とフォールバック

1. DWM角丸と一時ウィンドウ向け背景効果をWindows 11で技術検証します。
2. 効果が無効なWindows 11向けの単色フォールバックを実装します。
3. 非アクティブ時、リモートデスクトップ、高コントラスト時の背景を確認します。
4. アニメーションを追加する場合は、Windowsのアニメーション設定へ従います。

### Phase 5：検証と文書更新

1. 自動検証を実行します。
2. Windows 11の100%、150%、200% DPIで配置を確認します。
3. 下、上、左、右のタスクバーで表示位置を確認します。
4. 単一モニターと複数モニターで作業領域外へ出ないことを確認します。
5. マウス、キーボード、Narrator、外側クリックで開閉を確認します。
6. ライト、ダーク、高コントラストで状態を確認します。
7. Explorer再起動後と二重起動時の表示を確認します。
8. レビュー用スクリーンショットを保存し、本計画の完了条件と照合します。
9. `DESIGN.md`、`docs/TESTING.md`、ADRを実装結果へ合わせます。

## 自動テスト

- 通知アイコン矩形とフライアウトサイズから四辺の配置を計算できること。
- 100%、150%、200%相当のDPIで作業領域を越えないこと。
- 同じ通知アイコン操作で表示と非表示が切り替わること。
- Escapeと非アクティブ化が終了ではなく非表示を要求すること。
- 現在の電源側だけが初期展開されること。
- 反対側を適用しても未適用の選択を失わないこと。
- 未適用時だけ適用操作を表示すること。
- 未適用時は現在値と変更後の位置を別々に表示し、適用後は一つのつまみに戻ること。
- 左右キーで六つのプリセットを順番に移動し、端で範囲外へ進まないこと。
- 成功通知が期限後に消え、エラーは残ること。
- AutomationツリーでAC、バッテリー、各プリセット名を識別できること。
- 自動テストがPowrProf APIとPower Requestを実行しないこと。

## 実機確認

- Windows 11
- DPI 100%、150%、200%
- タスクバーの下、上、左、右
- 単一モニターと異なるDPIの複数モニター
- ライト、ダーク、高コントラスト
- マウス、キーボード、Narrator

- アプリ終了後の通知領域アイコン残留

## リスクと対処

### 非アクティブ化による早すぎる自動クローズ

スライダーのツールチップ、補助メニュー、スクリーンリーダー操作が別Popupを開くと、単純な`Deactivated`処理はフライアウトを閉じる可能性があります。
閉じる処理を遅延させ、所有Popupとフォーカスの移動先を確認します。

### Windows 11のBuildによる外観差

DWM角丸と一時ウィンドウ向け背景効果はWindows 11のBuild番号に依存します。
外観効果は機能要件にせず、単色フォールバックでも操作と可読性が成立するようにします。

### 永続設定の誤操作

コントロールセンターらしさを理由に、スライダー操作中の連続書込みは行いません。
未適用状態と小型の適用操作を明示し、読戻しが一致した場合だけ成功を表示します。

### 小型化によるアクセシビリティ低下

情報量を減らしても、文字、フォーカス、操作領域、実値の読み上げは縮小しません。
主要操作へ44×44 DIP程度の領域を確保し、200% DPIでは必要に応じて内容領域だけをスクロールさせます。

## 計画時点の判断

- 主画面は独立した設定ウィンドウではなく、通知領域の小型フライアウトとします。
- スリープ防止を最初の主要操作とします。
- 現在の電源側を既定で展開します。
- 消灯時間は即時書込みせず、変更した行だけ明示的に適用します。
- 右クリックメニューは補助操作へ縮小します。
- Windows 11の背景効果は技術検証後に採否を確定します。
- BatteryMonitorは視覚的な参照に限定し、WPF `Popup`とホバー中心の操作は採用しません。

## 実装結果

2026年7月15日に、BatteryMonitorを参照した視覚再設計と離散スライダーを実装しました。
主画面は372 DIPの小型WPF `Window`とし、一枚の外側サーフェス、小さい製品名、スリープ防止の主状態、独立したACとバッテリーのカードで構成しています。

離散スライダーはListBoxのAutomationを維持しながら、通常の目盛り、変更後のつまみ、適用済みの現在値リング、キーボードフォーカスリングを別々に描画します。
選択だけではWindowsへ書き込まず、未適用時だけ「変更後」と「適用」を表示します。

Windows 11では一時ウィンドウ向けDesktop AcrylicとDWM角丸を要求します。
高コントラストでは背景効果を無効にし、不透明なSystemColorsへ切り替えます。

`pwsh -NoProfile -File ./scripts/verify.ps1`は成功しました。
Coreテスト20件とAppテスト24件が成功し、警告とエラーはありません。
150% DPIの描画結果は確認済みですが、タスクバー四辺、100%と200% DPI、複数モニター、Narrator、ライトテーマ、高コントラストは未確認です。
