# DisplayLight MVP実装

## 目的

Codex連携を使わずに、通知領域からAC時とバッテリー時のディスプレイ消灯時間を変更し、手動でシステムスリープ防止を切り替えられるWindowsアプリを完成させます。

## 完了条件

- AC時とバッテリー時の現在値を表示できる。
- `1分`、`5分`、`10分`、`30分`、`60分`、`無期限`をスライダーで設定できる。
- 設定変更後の実値を読み直し、失敗を成功と表示しない。
- 手動スリープ防止を切り替えられ、ディスプレイ消灯は妨げない。
- AC電源限定を設定でき、バッテリー切り替え時に要求を解放する。
- 通知領域から画面表示、主要操作、終了を実行できる。
- 二重起動せず、二回目の起動で既存画面を表示する。
- 設定をユーザー単位で保存し、破損時に安全な既定値へ戻る。
- `pwsh -NoProfile -File ./scripts/verify.ps1`が成功する。
- 自動テストから実際の電源設定変更と電源要求を実行しない。
- 実機確認を実施していない項目を`docs/TESTING.md`に明記する。

## 対象外

- Codexおよび他のエージェントとの連携
- 複数エージェントの状態管理
- 作業完了後の自動スリープ
- 独自の低バッテリー下限
- 自動起動、インストーラー、自動更新

## 現在の設計

Coreは、プリセット、設定値、手動スリープ防止ポリシー、OS操作のインターフェースを持ちます。
Appは、WPF、通知領域、Windows電源API、設定ファイル、単一インスタンスを担当します。

ディスプレイ設定はPowrProf APIで読み書きし、変更後にAC値とDC値を読み直します。
スリープ防止はPower Request APIの`SystemRequired`だけを使用します。
どちらもWindows固有アダプターへ閉じ込め、自動テストから実APIを呼び出しません。

## 実装手順

1. 公式仕様を確認し、電源設定とPower Request APIのADRを作成します。
2. Coreへプリセット、設定、電源状態、スリープ防止ポリシーを追加し、単体テストを作成します。
3. Windows電源設定、Power Request、設定保存、電源状態監視、単一インスタンスを実装します。
4. ViewModel、WPF画面、通知領域を実装し、アプリのライフサイクルへ接続します。
5. format、Release build、単体テスト、プロセス起動確認を実行します。
6. README、DESIGN、ADR、手動テスト記録を実装結果へ合わせます。

## 発見と判断

- `Stop` Hookはターン停止であり製品上の完了ではないため、Codex連携をMVPから除外した。
- 消灯時間の変更は永続的なWindows設定とし、自動復元を行わない。
- 無期限はスライダーの右端に置き、カスタム入力をMVPへ含めない。
- 独自の低バッテリー下限を設けず、AC電源限定で利用者が制御する。
- 表示言語へ依存しない読取りと書込みのため、`powercfg`ではなくPowrProf APIへ統一した。
- スリープ防止はプロセス所有の理由付きハンドルを管理できるPower Request APIを採用した。
- プリセット外の現在値は実値を表示し、スライダーを近い値へ暗黙に変更しない。
- 通知領域は`Shell_NotifyIconW`を直接使用し、二重起動はユーザーとセッション単位のMutexとイベントで処理する。

- 画面からPower Requestの作成と解除が成功することを確認した。
- 直接依存と推移依存に更新候補および既知の脆弱なNuGetパッケージが報告されないことを確認した。
