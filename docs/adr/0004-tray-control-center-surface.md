# 0004 主画面を通知領域フライアウトへ変更する

- 状態：Proposed
- 日付：2026-07-15

## 文脈

ADR 0003では、MVPの主画面として通常のWPFウィンドウを採用しました。
実機で確認した結果、標準タイトルバー、中央表示、大型ヘッダー、常設フッターが残り、通知領域から開く小型コントロールセンターという製品像に合いませんでした。
電源操作の機能は成立しているため、電源APIを変更せず、表示面と開閉モデルを見直します。

## 提案

主画面を、通知アイコンの近くへ表示する固定幅のWPFフライアウトへ変更します。
フライアウトは標準タイトルバー、タスクバー表示、リサイズを持ちません。
左クリックで表示と非表示を切り替え、外側クリックとEscapeで閉じます。
通知アイコンの矩形は`Shell_NotifyIconGetRect`で取得し、モニターの作業領域とDPIに合わせて配置します。

スリープ防止を最初の主要操作とし、AC限定を従属設定として表示します。
ACとバッテリーの消灯時間は一つのセクションへ統合し、現在の電源側だけを既定で展開します。
ディスプレイ設定は永続変更であるため、スライダーのドラッグ中には書き込まず、未適用時だけ小型の適用操作を表示します。

## 影響

通常ウィンドウ向けの大型ヘッダー、独立カード、常設フッター、画面内の「隠す」と「終了」を削除します。
通知領域サービスは、表示要求だけでなく、通知アイコンの位置と入力種別をフライアウトへ渡す必要があります。
配置計算、フォーカス、外側クリック、DPI、高コントラストの検証項目が増えます。

電源API、Coreのポリシー、設定形式、単一インスタンス制御は維持します。
Windows 11で角丸と背景効果を利用できない場合は単色サーフェスを使用し、外観効果は失敗しても機能へ影響しない装飾として扱います。

## 比較した案

通常ウィンドウのサイズと余白だけを縮める案は、標準タイトルバー、中央配置、タスクバー表示、外側クリックで閉じない挙動が残るため採用しません。
通知領域の右クリックメニューだけで全操作を行う案は、二つの離散スライダー、現在値、未適用値、エラーを明確に表示できないため採用しません。
通常ウィンドウと小型フライアウトを両方持つ案は、MVPの設定量に対して操作面と状態同期が増えるため採用しません。

## 確定条件

技術スパイクでWindows 11、複数DPI、タスクバー四辺、キーボード起動を確認します。
確認後に本ADRをAcceptedへ変更し、ADR 0003の通常ウィンドウに関する部分を置き換えます。
