# 0006 Velopackによるインストールと自動更新

- 状態：Accepted
- 日付：2026-07-18

## 文脈

自己完結型ZIPは.NETランタイムを別途要求しませんが、利用者が多数のDLLを含むフォルダーを管理し、更新のたびに手作業で置き換える必要があります。
DisplayLightは通知領域へ常駐し、更新時には実行中プロセスとPower Requestを安全に終了させる必要があります。
既存のBatteryMonitorはVelopackとGitHub Releasesを使用しており、同じアカウントと配布基盤で運用できます。

## 判断

Windows 11 x64向けの配布と更新にVelopack 1.2.0を採用します。
NuGetパッケージとローカルCLIは同じ版をリポジトリで固定します。
アプリは自己完結・単一ファイルで発行し、VelopackがSetup、Portable ZIP、更新パッケージを生成します。

インストール版はGitHub Releasesの安定版だけを確認します。
起動から20秒後の確認では更新がある場合だけ通知し、手動確認では結果を常に表示します。
ダウンロードと再起動は利用者が画面上の操作を行った場合だけ開始します。
更新適用の直前に、ディスプレイ消灯カウントダウンとスリープ防止要求を解除します。

Portable ZIPでは更新の確認結果としてインストール版が必要であることを案内し、更新を適用しません。

## 結果

通常利用者は多数のDLLを直接管理せず、以後の安定版をアプリ内から導入できます。
最初のVelopack版より前のZIP利用者は、一度だけSetupへ手動で移行する必要があります。
公開GitHubリポジトリへの無認証アクセスにはGitHubのレート制限が適用されます。
コード署名を導入するまでは、Setupと更新バイナリにSmartScreenの警告が表示される可能性があります。
Portable利用者の設定をインストール版へ自動移行する専用処理は設けませんが、両者は同じローカル設定場所を使用します。

## 比較した案

自己完結型ZIPだけを継続する案は構成が単純ですが、手動更新と多数の配布ファイルを解消できないため採用しませんでした。
MSIXはWindows標準の配布境界を得られますが、現在のGitHub Releases中心の小規模配布と更新要件に対して導入範囲が大きいため見送りました。
独自更新機能は署名検証、差分取得、プロセス置換、ロールバックを自前で安全に実装する必要があるため採用しませんでした。
