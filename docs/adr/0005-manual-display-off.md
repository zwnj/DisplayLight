# 0005 手動ディスプレイ消灯をキャンセル可能にする

- 状態：Accepted
- 日付：2026-07-17

## 文脈

利用者が通知領域から接続中のディスプレイをすぐに消灯できる操作を追加します。
即時実行は誤操作を取り消せず、フライアウトを閉じた後も予約が残る動作は利用者の意図に反します。
この操作は、電源プランへ残る消灯時間設定や、プロセスが保持するスリープ防止要求とは異なる一回限りの操作です。

## 判断

「ディスプレイをオフ」は3秒のカウントダウン後に実行します。
同じボタンの再押下、Escape、外側クリック、フライアウトを閉じる操作でキャンセルします。
ゼロへ到達した場合は、フライアウトを閉じるモーションが完了してから一度だけ実行します。

Windows APIはMicrosoftの[`WM_SYSCOMMAND`仕様](https://learn.microsoft.com/en-us/windows/win32/menurc/wm-syscommand)にある`SC_MONITORPOWER`を使用し、`lParam=2`で消灯します。
ブロードキャストによる他プロセスの待機を避け、DisplayLight自身のウィンドウハンドルを`DefWindowProcW`へ渡します。

## 結果

ディスプレイ消灯はスリープ防止と独立して実行するため、システムを起動したまま画面だけを消す用途に使用できます。
電源プラン、スリープ防止要求、保存設定は変更しません。
カウントダウンとWin32呼び出しは分離し、自動テストでは実ディスプレイを消灯しません。

物理的な消灯と入力による再点灯は実機で確認する必要があります。
