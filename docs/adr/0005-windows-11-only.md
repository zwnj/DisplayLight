# 0005 対象OSをWindows 11へ限定する

- 状態：Accepted
- 日付：2026-07-15

## 文脈

初期設計はWindows 10とWindows 11を対象としていました。
二つのOSを維持すると、通知領域フライアウトの外観、DWM APIの利用可否、テスト環境、フォールバック実装を別々に管理する必要があります。
2026年7月15日、利用者はWindows 10対応を製品要件から外しました。

## 判断

DisplayLightの対象OSをx64のWindows 11へ限定します。
Windows 10向けの実装、フォールバック、手動テスト、配布確認は行いません。
Windows 11のBuildや利用者設定によってDWMの外観効果を利用できない場合に限り、単色サーフェスへ切り替えます。

## 結果

通知領域フライアウトの設計と検証をWindows 11へ集中できます。
README、設計、テスト計画、UI再設計計画からWindows 10向けの完了条件を削除します。
既存コードのターゲットフレームワークは本判断だけでは変更せず、次の実装作業でWindows 11の最小Buildを決めてから更新します。

Windows 10で偶然動作する可能性はありますが、互換性を保証せず、不具合を対応対象にしません。

## 比較した案

Windows 10とWindows 11の両方を維持する案は、利用者が必要としない互換実装と検証を継続するため採用しませんでした。
Windows 11専用コードを条件分岐で追加しながらWindows 10も最善努力で支える案は、対応範囲が曖昧になるため採用しませんでした。
