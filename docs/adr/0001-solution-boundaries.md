# 0001 CoreとWindowsアプリを分離する

- 状態：Accepted
- 日付：2026-07-15

## 文脈

本製品は、実機の電源状態を変更するWindows APIと、電源条件を評価する判断ロジックを扱います。
自動テストから電源操作を呼ばずに状態遷移を検証できる境界が必要です。

## 判断

ソリューションを`DisplayLight.Core`と`DisplayLight.App`へ分けます。
Coreは`net10.0`を対象とし、WPFとWindows APIへ依存しません。
Appは`net10.0-windows`を対象とし、UI、Win32、設定保存のアダプターと依存関係の組み立てを担当します。

Coreの単体テストは独立した`DisplayLight.Core.Tests`で実行します。
Windows固有アダプターのテストプロジェクトは、実装対象が生まれた時点で追加します。

## 結果

状態集約と安全ポリシーを、実機の電源状態を変えずにテストできます。
Windows固有処理がAppへ集まるため、UIとインフラストラクチャの分離はコード規約で維持する必要があります。

Appが大きくなった場合は、Windowsアダプターだけを別プロジェクトへ移せます。
初期段階では、空のプロジェクトを増やしません。

## 比較した案

単一WPFプロジェクトは初期ファイル数が少ない一方、判断ロジックがWPFやWin32へ結びつきやすいため採用しませんでした。

Application、Domain、Infrastructure、Presentationの四プロジェクト構成は境界が明確ですが、現時点の規模では依存関係と定型コードが先に増えるため採用しませんでした。
