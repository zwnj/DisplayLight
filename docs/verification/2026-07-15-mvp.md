# 2026年7月15日 MVP実機確認

## 環境

- OS：Microsoft Windows 11 Home 10.0.26200、x64
- .NET SDK：10.0.202
- スリープ方式：Modern Standby（S0低電力アイドル、ネットワーク接続）
- 実行形態：`dotnet run`相当の開発ビルド

## 起動と単一インスタンス

アプリを起動し、WPFウィンドウと通知領域APIの登録がエラーなく完了することを確認しました。
画面の「隠す」でウィンドウを非表示にした後、二回目のプロセスを起動しました。
二回目のプロセスは終了コード`0`で終了し、既存ウィンドウが再表示されました。
画面の「終了」をUI Automationから実行し、既存プロセスが終了コード`0`で終了しました。
レビュー修正後のReleaseビルドでも同じ起動と終了のスモークテストが成功しました。

通知領域アイコンの左右クリック、DPI別配置、Explorer再起動後の再表示、終了後の残骸は目視確認していません。

## ディスプレイ消灯時間

実行前の現在の電源プランでは、AC値とDC値がともに`120秒`でした。
AC側のスライダーから`1分`を適用し、ネイティブAPIでAC値が`60秒`になったことを確認しました。
DC側のスライダーから`1分`を適用し、ネイティブAPIでDC値が`60秒`になったことを確認しました。

確認処理の`finally`でAC値とDC値をともに`120秒`へ戻し、再読取りでも`120秒`であることを確認しました。
Windows設定画面との相互反映、制限ユーザー、Windows 10は未確認です。

## 手動スリープ防止

画面の「スリープ防止を開始」を実行し、表示が「有効（ディスプレイ消灯は許可）」へ変わることを確認しました。
続けて解除を実行し、表示が「無効」へ戻ることを確認しました。
作成または解除APIが失敗した場合はUIが有効表示へ遷移しないため、この操作でAPI呼び出しの成功までは確認できています。

`powercfg /requests`によるOS側一覧の確認は、管理者権限が必要な環境だったため実施できませんでした。
アイドル待機、実際のディスプレイ消灯、明示的なスリープ、ACとバッテリーの物理切り替え、Modern Standbyの長時間動作は未確認です。
